SpringBoot实现图片上传功能

环境依赖

  • SpringBoot2.X
  • mysql+mybatis

功能实现代码

  • 图片上传的service代码

    • 通过接受前端传输的MultipartFile类型的图片,通过调用transferTo()方法将文件写入主机的文件目录中。

    • UUID用来生成一个商品的ID用于存入数据库，并将商品描述图片的文件路径存入数据库。

    • MultipartFile类中的getOriginalFilename方法用来获取文件名。

    @Service
public class UploadService {
    @Resource
    GoodsImageMapper goodsImageMapper;
    @Resource
    GoodInfoMapper goodInfoMapper;
    public void upLoadImageService(MultipartFile image,String goodsName, Double goodsPrice, String goodsDescribe, HttpSession session){
        Integer goodsId= UUID.randomUUID().toString().replace("-","").hashCode();
        if(image.isEmpty()){
            System.out.println("文件为空");
        }
        String fileName=image.getOriginalFilename();
        String suffixName=fileName.substring(fileName.lastIndexOf("."));
        fileName= goodsId+suffixName;
        File dest=new File("/Users/jim.chou/Desktop/Java/"+fileName);
        if (!dest.getParentFile().exists()){
            dest.getParentFile().mkdir();
        }
        try {
            image.transferTo(dest);
        }catch (IOException e){
            e.printStackTrace();
        }
        try {
            goodInfoMapper.publishGoods(goodsId,goodsName,goodsDescribe,goodsPrice, (String) session.getAttribute("username"));
            goodsImageMapper.UploadImage(goodsId,"/image/"+fileName);
        }catch (Exception e){
            e.printStackTrace();
        }
    }
}

  • 建立文件的映射关系

    • 为了主机文件文件目录和SpringBoot项目下的文件目录建立对应关系，需要在WebMvcConfigurer的实现类中做如下配置：

      @Configuration
public class MyMVCConfig implements WebMvcConfigurer {
    @Override
    public void addResourceHandlers(ResourceHandlerRegistry registry) {              registry.addResourceHandler("/image/**").addResourceLocations("file:/Users/jim.chou/Desktop/Java/");
    }
}

    • 这样就能将项目项目结构下的/image/**目录与机的/Users/jim.chou/Desktop/Java/目录建立对应关系

    multipart:
      max-file-size: 100MB
      max-request-size: 100MB
